# 접근성을 갖춘 2048 게임 개발 문서

## 1. 프로젝트 목표

이 프로젝트의 목표는 시각적인 정보 없이 소리와 키보드만으로 완벽하게 즐길 수 있는 '2048' 게임을 개발하는 것입니다. 특히 스크린리더 사용자가 게임의 모든 상황을 명확히 인지하고, 불편함 없이 전략적인 플레이를 할 수 있도록 만드는 데 중점을 둡니다.

## 2. 기술 스택

- **프레임워크:** React (Create React App)
- **언어:** JavaScript (ES6+)
- **스타일링:** CSS

## 3. 핵심 기능 및 설계 결정

이 게임은 일반적인 2048 게임과 달리, 시각장애인 사용자의 경험에 맞춰 다음과 같은 독창적인 기능과 설계가 적용되었습니다.

### 3.1. 이중 키보드 인터페이스 (Dual Keyboard Interface)

사용자와의 논의를 통해, '탐색'과 '실행'을 분리하는 대신, '확인'과 '실행'이라는 독특한 모델을 구축했습니다.

- **타일 이동 (화살표 키):**
  - `↑, ↓, ←, →`
  - 게임의 핵심 조작계로, 모든 타일을 해당 방향으로 움직이는 직관적인 역할을 수행합니다.

- **타일 즉시 확인 (키보드 격자):
  - `1,2,3,4`, `Q,W,E,R`, `A,S,D,F`, `Z,X,C,V`
  - 키보드 좌측의 4x4 키 배열을 게임 보드의 4x4 타일과 일대일로 매핑했습니다.
  - 사용자는 커서를 움직여 탐색할 필요 없이, 궁금한 위치의 키를 누르는 즉시 해당 칸의 숫자 값("16")이나 상태("빈 칸")를 들을 수 있습니다. 이는 순차적 탐색보다 훨씬 빠르고 효율적인 사용자 경험을 제공합니다.

### 3.2. 상세하고 명확한 음성 안내

모든 게임 상황은 스크린리더의 `aria-live` 영역을 통해 사용자에게 명확하게 전달됩니다.

- **이동 및 합치기 결과:** "오른쪽으로. 4와 4가 합쳐져 8이 됨. 8점 획득."과 같이 한 번의 조작으로 발생한 모든 이벤트를 종합하여 하나의 문장으로 안내합니다.
- **새로운 타일 위치 안내:** "F 위치에 새로운 타일 2 등장."과 같이, 새로 생성된 타일의 위치를 키보드 격자 키로 정확히 알려주어 사용자가 보드를 다시 탐색해야 하는 수고를 덜어줍니다.
- **반복 안내 문제 해결:** 동일한 값을 가진 타일 키를 반복적으로 눌러도 스크린리더가 매번 안내하도록, 메시지 끝에 보이지 않는 공백(`\u00A0`)을 번갈아 붙이는 'Alternating Space' 트릭을 적용하여 안정적인 피드백을 보장합니다.

### 3.3. 포커스 상실 문제 해결 (`Escape` 키 토글)

스크린리더 사용자가 페이지의 다른 부분(예: 안내문)을 탐색하기 위해 포커스를 옮겼을 때 게임 조작이 중단되는 문제를 해결했습니다.

- **전역 이벤트 리스너:** 키보드 이벤트를 특정 컴포넌트가 아닌 `window` 전체에서 감지하여, 포커스 위치와 상관없이 항상 게임 조작이 가능하도록 했습니다.
- **게임 활성/비활성 토글:** `Escape` 키를 누르면 '게임 조작 모드'를 켜고 끌 수 있습니다.
  - **활성화 (기본):** 모든 게임 관련 키가 정상 작동합니다.
  - **비활성화:** 게임 키는 잠시 꺼지고, 화살표 키는 스크린리더의 페이지 탐색 기능으로 동작합니다.
  - 상태가 변경될 때마다 "게임 조작 활성화/비활성화"를 음성으로 안내하여 사용자가 현재 모드를 명확히 알 수 있습니다.

### 3.4. 포괄적인 게임 가이드

사용자와의 지속적인 피드백을 통해, 시각장애인의 입장에서 궁금할 수 있는 모든 규칙을 상세히 설명하는 '완벽 가이드'를 게임 내에 포함시켰습니다. (타일 이동 방식, 합치기 규칙, 새로운 타일 생성 규칙, 최종 목표 등)

## 4. 로컬에서 실행하는 방법

```bash
# 1. 프로젝트 클론 후 디렉토리로 이동
cd 2048

# 2. 의존성 패키지 설치
npm install

# 3. 개발 서버 실행
npm start
```
실행 후 `http://localhost:3000` 에서 애플리케이션을 확인할 수 있습니다.

## 5. 향후 개발 고려사항

- **다국어 및 키보드 레이아웃 지원:** 현재 타일 확인 키맵은 QWERTY 키보드 레이아웃에 하드코딩되어 있습니다. 다른 키보드 레이아웃(Dvorak 등)을 사용하거나 다른 언어를 사용하는 사용자를 위해 키맵을 설정할 수 있는 옵션을 추가할 수 있습니다.
- **모바일 접근성:** 현재는 데스크톱 키보드 사용을 전제로 합니다. 모바일 스크린리더(TalkBack, VoiceOver for iOS) 사용자를 위해, 스와이프 제스처를 이용한 새로운 조작 방식을 추가로 개발할 수 있습니다.
- **설정 기능:** 사용자가 보드 크기, 색상 테마, 애니메이션 속도 등을 직접 설정할 수 있는 기능을 추가하여 개인화된 경험을 제공할 수 있습니다.

## 6. 온라인 게임으로의 발전 아이디어

이 게임을 온라인 서비스로 발전시킬 때, 시각장애인 사용자분들이 더욱 재미있게 즐길 수 있도록 다음과 같은 요소들을 고려할 수 있습니다.

### 6.1. 강화된 오디오 피드백 및 사운드 디자인
- **차별화된 효과음:** 타일이 미끄러질 때, 합쳐질 때, 새로운 타일이 나타날 때 등 각 상황에 맞는 독특하고 만족스러운 효과음을 추가합니다. 합쳐지는 타일의 숫자가 커질수록 효과음의 깊이나 강도를 다르게 하여 청각적인 쾌감을 높일 수 있습니다.
- **동적인 배경 음악:** 게임 보드가 채워지거나 점수가 높아질수록 배경 음악의 분위기나 속도가 변하여 긴장감과 몰입감을 더합니다.
- **음성 안내의 다양화:** 현재의 기계적인 음성 안내 외에, 친근하거나 개성 있는 목소리로 게임 상황을 알려주는 옵션을 제공할 수 있습니다.

### 6.2. 멀티플레이어 및 소셜 기능
- **경쟁 모드:** 두 명의 플레이어가 각자의 보드에서 동시에 플레이하여, 먼저 2048 타일을 만들거나 정해진 시간 안에 더 높은 점수를 얻는 사람이 승리하는 방식입니다. 상대방의 점수나 진행 상황을 음성으로 간략히 알려줄 수 있습니다.
- **협동 모드:** 두 명의 플레이어가 하나의 보드를 공유하며 번갈아 가며 플레이합니다. 서로의 전략을 논의하고 협력하는 재미를 줄 수 있습니다.
- **관전 모드:** 다른 플레이어의 게임을 음성 안내를 통해 관전할 수 있는 기능을 제공합니다.
- **리더보드:** 전 세계 또는 친구들 간의 최고 점수 순위를 제공하여 경쟁심을 자극합니다.

### 6.3. 개인화 및 성장 요소
- **사운드 팩/음성 팩:** 다양한 테마의 효과음이나 음성 안내 목소리를 선택할 수 있도록 하여 개인의 취향을 반영합니다.
- **도전 과제 및 업적:** 특정 점수 달성, 특정 타일 합치기, 연속 승리 등 다양한 도전 과제를 제공하고, 달성 시 보상(새로운 사운드 팩 등)을 제공합니다.
- **일일/주간 챌린지:** 매일 또는 매주 특별한 규칙이나 목표를 가진 챌린지를 제공하여 꾸준한 참여를 유도합니다.

### 6.4. 튜토리얼 및 가이드 플레이
- 초보자를 위한 인터랙티브 오디오 튜토리얼을 제공하여, 게임 규칙과 조작법을 소리만으로 쉽게 익힐 수 있도록 돕습니다.
